Background & Motivation

  • Common intuition: December's holiday spending dwarfs January's post-holiday lull.
  • Retailers and card issuers act on that assumption — but rarely test it on transaction-level data.
  • This project tests whether the gap is real, using both classical and computational inference.

Why it matters

If the December effect is real, it shapes staffing, stock, and promotions. If not, those decisions rest on a story the data doesn't tell.

Research Question

Is the average credit-card transaction amount in December 2019 significantly different from January 2020?
  • Two-sided by design — we want to detect a difference in either direction, not assume December must be larger.
  • Outcome of interest: mean transaction amount (USD), not transaction count or category mix.

Hypotheses

Null Hypothesis · H0
μD = μJ The true mean transaction amounts are equal in December and January.
Alternative · Ha
μD ≠ μJ The true mean transaction amounts differ between the two months.
  • Significance level: α = .05, two-tailed.
  • μD = population mean transaction amount, December 2019; μJ = population mean transaction amount, January 2020.
  • Reject H0 only if observed data are unlikely (< 5%) under the null.

Computational Inference · Bootstrap

Procedure

  1. Resample December and January with replacement, preserving group sizes.
  2. Compute δ̂ = X̄Dec − X̄Jan on each resample.
  3. Repeat B = 10,000 times → bootstrap distribution of the mean difference.
  4. 95% percentile CI from the 2.5th and 97.5th percentiles; p-value via null-shifted resamples.

Why bootstrap here

  • Data is severely right-skewed (skew = 70.98) — bootstrap needs no distributional assumption.
  • Acts as an independent check on the t-test, not just a re-derivation of it.

Which Result I Lean On

I lean on the bootstrap as the primary result, with the t-test as confirmatory.

Statistical reasoning

  • Data is severely right-skewed (skew = 70.98 in December) — exactly where parametric tests are riskiest.
  • Bootstrap assumes nothing about the distribution and doesn't lean on the CLT.
  • The t-test is valid here only because n is enormous; bootstrap is the safer default for skewed financial data.
  • Because both methods agree, the t-test serves as independent confirmation.

Conclusion · So What?

Statistical bottom line

No significant difference between December and January (t-test p = .121; bootstrap p = .128; 95% CI ≈ [−$0.29, $2.51]). The $1.11 gap is noise.

Practical / managerial implications

  • If a holiday effect exists, it likely shows up in transaction frequency, not amount per swipe.
  • Retailers chasing higher per-transaction revenue in December may be optimizing the wrong lever.
  • Caveat: data is simulated — replicate on real transactions before acting on it.
  • Method takeaway: pairing classical + computational inference guards against single-assumption blind spots.
